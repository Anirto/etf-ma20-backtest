# ETF MA20 仓位管理策略 · 量化回测

> 一个基于 20 日均线的 ETF 仓位管理策略，通过价格与均线的相对位置动态调整仓位。

---

## 📖 策略规则

```
┌─────────────────────────────────────────────────────┐
│  价格 > MA20（20日均线）→ 卖出当前仓位的 20%        │
│  价格 < MA20（20日均线）→ 加仓（买入）当前资金的20% │
└─────────────────────────────────────────────────────┘
```

### 核心思想

- **趋势跟随**：当 ETF 价格运行在均线上方时，表明处于上升趋势，逐步兑现利润
- **逆势加仓**：当 ETF 价格跌到均线下方时，认为是相对低位，分批买入摊低成本
- **动态调整**：每次只调整 20% 仓位，避免一次性重仓或清仓带来的冲击

### 交易参数

| 参数 | 值 |
|:---|:---|
| 均线周期 | MA20 |
| 调整比例 | 每次 20% |
| 佣金费率 | 万分之三（最低 5 元） |
| 印花税 | 千分之一（仅卖出） |

### 分析结论

1. **宽基 ETF 表现稳健**：上证 50、沪深 300 等大盘指数回撤控制在 9% 以内，夏普 > 1.0，适合低风险偏好
2. **成长风格收益更高**：创业板、科创 50 年化 13%-17%，但回撤也相对较大（15%-19%）
3. **证券 ETF 表现最差**：证券板块在回测期内持续震荡，MA20 策略在震荡市中容易被反复摩擦产生交易成本
4. **行业 ETF 分化明显**：军工 +31.9%，证券 -9.6%，行业选择对策略效果影响巨大
5. **胜率偏高需辩证看待**：胜率 98%+ 是因为每次卖出价基本都高于最近买入价（均值回复特性），但部分 ETF 总收益仍然为负（如证券 ETF）

## 🚀 使用方法

### 环境要求

- Python 3.8+
- 网络连接（需访问东方财富接口）

### 安装依赖

```bash
pip install -r requirements.txt
```

> **注意**：AkShare 依赖 `requests`，如果你的系统配置了代理（如 Clash），需要在运行前关闭系统代理或设置 `NO_PROXY=*`，否则会连接失败。

### 运行回测

```bash
cd etf-ma20-backtest/
python scripts/backtest_etf_ma20.py
```

### 输出文件

```
etf-ma20-backtest/
├── scripts/
│   └── backtest_etf_ma20.py   # 回测脚本
├── data/
│   └── etf_ma20_results.json   # 回测结果（JSON 格式）
├── requirements.txt            # Python 依赖
├── .gitignore
└── README.md
```

回测结果以 JSON 格式保存在 `data/etf_ma20_results.json`，包含：
- 每只 ETF 的完整绩效指标
- 所有交易记录（日期、方向、价格、数量）
- 每日净值曲线（可用于后续绘图分析）
